## Overview

This repository hosts the **GCO (Grants and Contracts Officer) Needs Assessment** tool, an interactive HTML form used during onboarding to help incoming GCOs self-assess their comfort and familiarity across the full spectrum of pre- and post-award research administration tasks.

The tool is embedded in the department's Notion-based onboarding and syllabus page and is served via GitHub Pages.

## What the Tool Covers

### Pre-Award
- **Track A — NOFO-Based Proposal Submissions (Grants)**
  - NOFO analysis, LSO coordination, DSS/eProposal, budget development, clinical trial pre-award, IRB/IACUC/IBC, review and submission, post-submission
- **Track B — Investigator-Initiated (Agreements/Contracts)**
  - Contract mechanism analysis, agreement types (CTA/CDA/DUA/BAA/ISAF), sponsored and non-sponsored agreement development
- **Sponsor Landscape Familiarity**
  - Federal sponsors (NIH ICs, NSF, DOD/MTEC, SAMHSA, HRSA), state sponsors (UT HHS, UT OME), NNDC, and foundations

### Post-Award
- Award receipt and setup
- Financial management and monitoring
- Award modifications and prior approvals
- Compliance and reporting
- Extensions (NCE) and carryover
- Supplements
- Subaward management
- Clinical trial post-award
- Award closeout

---

## How It Works

1. The incoming GCO opens the tool in a browser via the Notion onboarding page or the direct link above
2. They enter their name and rate each sub-task using color-coded circles:
   - 🟢 Comfortable / Familiar
   - 🟡 Some experience / Some exposure
   - 🔴 Need training / Not familiar
3. Each section displays a **worst-case rollup dot** reflecting the lowest rating given among its sub-tasks
4. When finished, the GCO clicks **Copy to clipboard** (separately for pre- and post-award) and pastes the results into the designated area of the Notion onboarding page
5. The manager reviews the pasted results to identify training priorities

## Updating the Tool

To revise task lists, add sponsors, or adjust section structure:
1. Edit `GCO_needs_assessment_v1.html` locally or directly in GitHub
2. Commit changes to the `main` branch
3. GitHub Pages will republish automatically within 1–2 minutes
4. The Notion embed URL does not need to be updated as long as the filename stays the same
